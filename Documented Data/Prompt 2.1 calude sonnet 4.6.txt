## ⚔️ CORE SYSTEM LOGIC (MANDATORY)

1. **Public Access**

   * Landing page is fully accessible without authentication
   * Users can view all available services freely

2. **User Model**

   * Every new user is a **CUSTOMER by default**
   * No elevated privileges without admin action

3. **Service Request Flow**

   * Users can request access to services
   * Requests are reviewed by ADMIN

4. **Admin Authority**

   * ADMIN has full control:

     * Approves users
     * Assigns users into families
     * Sends invites
     * Assigns ORGANIZER role

5. **Organizer Role**

   * Limited scope:

     * Can only manage assigned families
     * Can request new families/services
     * Cannot override admin decisions

6. **Invite System (STRICT ENFORCEMENT)**

   * Invites must:

     * Be assigned to ONE specific user
     * Be created ONLY by ADMIN
     * Be linked to a family and a service
     * Be **single-use only**

   * Invite lifecycle:

     * PENDING → ACCEPTED / DECLINED / EXPIRED
     * Once ACCEPTED → cannot be reused or modified

   * Must prevent:

     * Duplicate active invites
     * Inviting users already in a family

7. **Membership Activation**

   * A user becomes an active member ONLY after:

     * Accepting an invite
     * Being assigned to a family
